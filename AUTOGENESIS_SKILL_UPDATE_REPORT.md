# AutoGenesis 技能重要更新 - 禁止模拟录制

**更新时间**: 2026-04-03 13:59 GMT+8  
**更新原因**: 在实际执行中发现模拟录制的严重问题  
**影响级别**: 🔴 **关键更新** - 涉及技能核心原则

---

## 🚫 **新增核心原则：严禁模拟录制**

### ❌ **发现的问题**
在今天的 AutoGenesis 执行过程中，尝试通过模拟的方式生成步骤定义：
- 手写基于假设的 XPath 定位器
- 编造 MCP 工具调用代码
- 推测元素属性和页面状态

### 💥 **导致的后果**
- 生成的步骤定义不准确，无法在真实环境中执行
- 无法感知应用状态变化（如窗口关闭、失去焦点）
- 违背了 AutoGenesis 自动化录制的核心价值

### ✅ **解决方案**
在 SKILL.md 中新增以下关键内容：

#### 1. **严禁模拟录制规则**
```markdown
🚫 严禁模拟录制（核心原则）
AutoGenesis 的核心价值在于通过真实的 MCP 交互获得准确的元素定位和步骤定义。

绝对禁止的做法：
❌ 模拟生成步骤定义 - 手写或推测任何 @step 函数代码
❌ 假设元素定位 - 不通过实际页面结构猜测 XPath、ACCESSIBILITY_ID 等
❌ 编造工具调用结果 - 不模拟 MCP 工具的返回值
❌ 跳过真实验证 - 不基于推测判断步骤是否成功
```

#### 2. **真实案例教训**
```markdown
🔍 真实案例教训
在 2026-04-03 的执行中发现了模拟录制的严重问题：
- 尝试模拟生成 "Pin and unpin a tab" 场景的步骤定义
- 手写了基于假设的 XPath 定位器和工具调用代码  
- 结果：当尝试真实验证时发现 Edge 窗口已关闭，页面状态完全不符合预期
- 教训：只有通过真实 MCP 交互才能确保步骤定义的准确性和可靠性
```

#### 3. **更新录制流程**
强调每个步骤都必须：
- 调用真实的 MCP 工具
- 检查返回结果
- 基于真实页面状态调整策略
- 使用实际的元素定位器
